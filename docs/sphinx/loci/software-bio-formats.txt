Bio-Formats
===========

Overview
--------

Bio-Formats is a standalone Java library for reading and writing life
sciences image file formats. It is capable of parsing both pixels and
metadata for a large number of formats, as well as writing to several
formats. See the table below for a complete list (click the headers to
sort, and format names to see all information). For more information on
the format ratings, see the `Supported Formats </bio-formats/formats>`_
page.

Purpose
-------

****\ Bio-Formats's primary purpose is to convert proprietary microscopy
data into an open standard called the `OME data
model <http://genomebiology.com/2005/6/5/R47>`_, particularly into the
`OME-TIFF <http://ome-xml.org/wiki/OmeTiff>`_ file format. See the
:doc:`Bio-Formats statement of purpose <bio-formats-about>` for a thorough
explanation and rationale.

Version history and roadmap
~~~~~~~~~~~~~~~~~~~~~~~~~~~

****\ Bio-Formats is updated whenever a significant bug fix is made, or
whenever a new version of
`OMERO <http://openmicroscopy.org/site/support/omero>`_ is released. The
version number is three numbers separated by dots; e.g., 4.0.0. See the
`what's
new <http://dev.loci.wisc.edu/svn/software/trunk/components/bio-formats/doc/whats-new.txt>`_
list for a history of major changes in each release.

The latest stable version of Bio-Formats is 4.3.3. For future
development directions, see the
`4.4 <http://trac.openmicroscopy.org.uk/ome/query?group=status&component=Bio-Formats&milestone=OMERO-Beta4.4>`_
and \ `5.0 <http://trac.openmicroscopy.org.uk/ome/query?group=status&component=Bio-Formats&milestone=OME-5.0>`_
roadmaps.
